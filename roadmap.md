# Роудмап для PWA D&D Билдера

Этот роудмап поможет тебе создать основу для PWA D&D билдера, начиная с HTML и двигаясь к CSS и JS. Отмечай галочки (`[x]`) по мере выполнения, чтобы видеть прогресс!

## Этап 1: Основа HTML (1-2 недели)
Цель: Создать "скелет" интерфейса билдера с семантическими тегами.

- [ ] **Создать базовую структуру страницы**
  - Добавить теги `<html>`, `<head>`, `<body>` в `index.html`.
  - В `<head>` добавить `<meta name="viewport" content="width=device-width, initial-scale=1.0">` для адаптивности на Android.
  - Проверить страницу через Live Server.

- [ ] **Добавить заголовок и подзаголовок**
  - Использовать `<h1>` для названия "D&D Билдер".
  - Добавить `<h2>` для подзаголовка, например, "Создай своего героя".

- [ ] **Создать форму для персонажа**
  - Добавить `<form>` с `<fieldset>` и `<legend>` (например, "Создание персонажа").
  - Включить `<input>` для имени персонажа.
  - Проверить, как выглядит на телефоне.

- [ ] **Добавить список рас**
  - Создать `<select>` с `<option>` для 3 рас (человек, эльф, гном).
  - Попробовать `<ul>` или `<ol>` с `<li>` для списка рас (для тренировки).
  - Запомнить: `<ul>` — ненумерованный, `<ol>` — нумерованный, `<li>` — элемент списка.

- [ ] **Использовать семантические теги**
  - Добавить `<section id="character-form">` для формы.
  - Попробовать `<article>` для блока результатов (например, будущих статов).
  - Проверить семантику через DevTools (F12, вкладка "Elements").

- [ ] **Закрепить списки**
  - Написать `<ul>` и `<ol>` с 3 элементами `<li>` (например, классы: воин, маг, жрец) без подсказок.
  - Если забыл, заглянуть в гайд и записать комментарий в HTML: `<!-- ul - ненумерованный список -->`.

## Этап 2: Основы CSS (2-3 недели)
Цель: Сделать билдер красивым и адаптивным для Android.

- [ ] **Подключить CSS**
  - Создать файл `styles.css` и подключить в `<head>` через `<link>`.
  - Добавить стиль для `<h1>` (например, цвет или шрифт).

- [ ] **Стилизовать форму**
  - Добавить рамку для `<fieldset>` (как тебе понравилось с `<legend>`).
  - Сделать `<input>` и `<select>` шире или с отступами.

- [ ] **Попробовать строчное расположение**
  - Поставить два `<section>` или `<article>` в строку (как ты хотел).
  - Использовать `display: inline-block` или `display: flex` для контейнера.

- [ ] **Сделать адаптивность**
  - Проверить, как форма выглядит на телефоне через DevTools (F12, режим телефона).
  - Добавить `max-width: 100%` для элементов, чтобы не ломались на Android.

- [ ] **Попробовать Tailwind (опционально)**
  - Подключить Tailwind CSS через CDN.
  - Добавить классы вроде `border` или `flex` для `<section>`.

## Этап 3: Первые шаги в JS (3-4 недели)
Цель: Добавить интерактивность для генерации персонажа.

- [ ] **Подключить JS**
  - Создать файл `script.js` и подключить в `<body>` через `<script>`.
  - Написать `alert("Билдер работает!")` для теста.

- [ ] **Взять данные из формы**
  - Получить значение из `<input>` (имя персонажа) через JS.
  - Вывести имя в `<p>` или через `alert`.

- [ ] **Сгенерировать статы**
  - Написать функцию для броска 4d6 (убрать меньший).
  - Вывести результат в `<section>` или `<article>`.

- [ ] **Закрепить списки в JS**
  - Создать массив рас (например, `["человек", "эльф", "гном"]`) и вывести в `<ul>` через JS.
  - Это поможет запомнить `<ul>`, `<ol>`, `<li>`.

## Этап 4: Превратить в PWA (2-3 недели)
Цель: Сделать билдер устанавливаемым на Android.

- [ ] **Добавить manifest.json**
  - Создать файл `manifest.json` с именем и иконкой приложения.
  - Подключить в `<head>` через `<link>`.

- [ ] **Настроить Service Worker**
  - Создать `sw.js` для кэширования `index.html` и `styles.css`.
  - Проверить офлайн-режим в браузере.

- [ ] **Протестировать на Android**
  - Открыть страницу на телефоне и добавить на главный экран.
  - Проверить, что билдер выглядит как приложение.
